# Project Reader 技能

当用户要求阅读项目、解释项目结构、分析某个代码文件、总结后端或前端流程时，使用本技能。

## 处理步骤

1. 先调用 `list_files` 查看项目目录，不要凭空猜测。
2. 如果用户指定某个文件，再调用 `read_file` 读取真实内容。
3. 解释项目时按以下结构：
   - 这个文件/目录负责什么
   - 它和其他文件怎么连接
   - 关键函数或类是什么
   - 数据流是怎么走的
   - 新手最容易误解哪里
4. 如果代码较长，不要逐行机械解释，优先解释核心流程。
5. 如果用户说“我看不懂”，要降低难度，用生活类比解释。

## 回答模板

可以按这个顺序回答：

1. 这个文件的作用
2. 核心流程
3. 关键代码解释
4. 和项目其他部分的关系
5. 你接下来应该看哪里
